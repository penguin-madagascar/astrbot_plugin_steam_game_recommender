# 更新日志

## 未发布

- 新增 `/accountbind`（alias `/账号绑定`），支持为当前聊天用户绑定 SteamID64 或 Steam 好友码，并使用 SQLite 保存可扩展账号绑定记录。
- `/gamerec` 支持 `排除已有` / `exclude-owned` 和 `仅查看已有` / `only-owned` 两个互斥库过滤参数。
- 新增 `/unplayedrec`（alias `/未玩推荐`），可从已绑定 Steam 游戏库中随机返回一款未游玩且 Steam 评价过线的游戏。
- `/gamerec` 默认改为严格匹配排序；多样性模式改由 LLM 从用户描述中判断，失败或非法值回退严格匹配。
- 新增 `/gamerec_retry`（alias `/重新推荐`、`/换一批`），可基于最近 30 分钟内的 `/gamerec` 结果排除已展示游戏并换一批。
- 新增 `steam_api_key` 配置，用于 Steam Web API `GetOwnedGames`；未配置或游戏库不可见时，库过滤和未玩推荐会直接提示错误。
- 新增 `enable_llm_fallback` 配置；开启后，空结果或超出 Steam/PC 覆盖范围时可生成已标注的 LLM 未验证建议。

## 0.3.2 - 2026-06-29

- 将推荐流程改为 Steam-only：所有 `/gamerec` 请求默认走本地 Steam 索引、标签归一和相似度排序。
- LLM 偏好解析新增额外标签和相似游戏提取，参考游戏标签来自 Steam 搜索结果，不再依赖手写画像。
- 跨平台请求会明确提示当前版本仅覆盖 Steam/PC。
- 将 `/gamerec` 输出拆分为一条开场白消息 + 每个推荐游戏一条消息。
- 将拆分后的推荐消息包装为一个新的合并转发聊天记录发送，避免在群聊内刷出多条普通消息。
- 改进推荐解释：输出具体适合点、平台 caveat 和明确不确定项，不再使用空泛的“暂未发现明显不适合点”。

## 0.3.1 - 2026-06-28

- 改进关键词偏好兜底：覆盖平台、双人、预算、语言、恐怖排除、难度和参考游戏线索。
- 强化 DLC、纯单人候选、重复版本和不喜欢类型的过滤与去重。
- 通过可选的 sibling 插件 `astrbot_plugin_steam_price_heybox` 增加 Steam 价格增强。
- 修复推荐理由格式化，避免带斜杠的口碑文本被错误拆分。

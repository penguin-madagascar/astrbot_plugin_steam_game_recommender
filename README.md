# Steam 游戏推荐助手

这是一个面向 AstrBot 的 Steam 游戏推荐插件。你可以直接描述想玩的玩法、参考游戏、语言、预算、开发商或其他具体要求，插件会给出推荐理由，并在资料不足或条件无法确认时主动说明。

普通推荐不需要 Steam Web API Key。只有读取个人游戏库、排除已拥有游戏或使用随机库内推荐时，才需要绑定 Steam 账号并配置 Key。

## 安装与配置

1. 在 AstrBot 插件管理中安装本插件；也可以把项目目录以 `astrbot_plugin_steam_game_recommender` 为名放入 AstrBot 的插件目录。
2. 手动安装时，在插件目录执行：

   ```bash
   pip install -r requirements.txt
   ```

   请确保依赖安装到 AstrBot 实际使用的 Python 环境中。

3. 在 AstrBot Dashboard 中按需设置：

   - 自然语言理解与推荐理由模型：留空时使用当前会话模型；模型暂不可用时仍会继续推荐，但复杂要求和理由可能更简略。
   - 无结果时的灵感推荐模型：默认关闭，只在 Steam 查询正常但没有合适结果时提供额外灵感。
   - 默认价格地区：未在命令中指定地区时使用，初始为 `CN`。
   - 每次最多推荐几款：新安装默认为 10，范围为 1–10。
   - 复用完全相同的推荐结果：默认关闭；开启后，相同需求通常会更快返回，并减少重复请求。
   - 随机推荐门槛：只影响 `/randomrec` 所要求的最低评测数和好评率。
   - Steam Web API Key：普通推荐不需要，游戏库过滤和 `/randomrec` 才需要。请妥善保管 Key，可前往 [Steam Web API Key 申请页](https://steamcommunity.com/dev/apikey) 获取。

如需当前价格、历史最低和最近促销，请在插件市场安装“Steam 价格查询（小黑盒）”（`astrbot_plugin_steam_price_heybox`）。未安装时仍可正常推荐，只是不会显示完整价格信息。

## 开始使用

### 命令一览

| 命令 | 中文命令 | 用途 |
| --- | --- | --- |
| `/gamerec <需求>` | `/游戏推荐 <需求>` | 根据自然语言需求推荐 Steam 游戏。 |
| `/gamerec_retry [补充要求]` | `/重新推荐`、`/换一批` | 在最近一次推荐基础上换一批，或补充、修改条件。 |
| `/accountbind <SteamID64 或好友码>` | `/账号绑定` | 绑定 Steam 账号；不带参数时查看当前绑定。 |
| `/randomrec` | `/随机推荐` | 从已绑定游戏库中随机挑选一款未玩且评价达到门槛的游戏。 |

`/gamerec` 的完整形式是：

```text
/gamerec [排除已有|仅查看已有] [区域] <自然语言需求>
```

建议把 `排除已有` 或 `仅查看已有` 写在需求开头，两者不能同时使用。区域可以写成 `-CN`、`-US`、`-JP` 等两字母代码，也可以写国区、美区、日区、港区、台区、韩区等常用名称。

例如：

```text
/gamerec 排除已有 -US 双人合作解谜，预算 30 美元
/游戏推荐 仅查看已有 日区 适合周末通关的剧情游戏，预算 3000 日元
/gamerec 类似黑暗之魂的游戏
/gamerec 推荐 3 款支持简体中文、预算 100 元以内的合作游戏
/gamerec 模拟类游戏，要有节日活动，最好由 ConcernedApe 开发
/gamerec 即将发售的类魂游戏
```

### 推荐数量

- 新安装默认最多返回 10 款。
- 用户原文中明确写出的数量优先，例如“推荐 3 款”会请求 3 款；支持阿拉伯数字和一至十的中文数字。
- 用户要求的数量不会超过 Dashboard 中保存的上限，也不会超过 10。比如上限为 5 时，“推荐 8 款”最多返回 5 款。
- 如果升级前已经设置为 5 款，升级后会保留 5，不会强行改成 10。
- `/gamerec_retry` 默认沿用上一次的数量，只有补充要求明确修改数量时才会变化。

目标数量不是凑数要求。符合条件的游戏不足时，实际返回数量会更少。

### 重复查询

插件默认关闭相同查询结果复用。关闭时，即使连续发送完全相同的需求，也会重新寻找和检查游戏；开启后，相同需求通常会更快返回，并减少网络和模型请求。

无论是否开启，插件仍会保留部分基础游戏资料，以避免每次查询都从头加载。

## 怎样提出需求

### 找“类似某款游戏”的作品

```text
/gamerec 类似黑暗之魂的游戏
/gamerec 类似逃离塔科夫，但不要 PVP
```

插件会尝试识别中文名、英文名和常见本地化名称，再结合参考游戏的玩法和 Steam 资料寻找相似作品。名称没有找到或存在歧义时，结果会提示你，并继续使用其他明确条件。

参考游戏本身不会作为结果返回，但续作和同系列作品仍可参与推荐。基础版、重制版和续作会分别判断，尽量避免把标题相近的另一部作品认错。

### 玩法、排除项和大作倾向

你可以同时写多个条件，并明确哪些是核心要求：

```text
/gamerec 必须本地双人合作，轻松解谜，不要恐怖
/gamerec 单机 3A 大作，支持手柄
```

明确的核心玩法比普通类型和热度更重要。插件不会用完全没有核心玩法直接证据的候选凑数；部分核心特征证据不足的候选仍可能返回，但会在“不推荐理由”中说明。条件较严格时，宁可少返回，也不会只因为某款游戏热门就加入结果。

“AAA”“3A”或“大作”表示偏好知名作品，插件会更重视口碑和关注度，但不会据此猜测动作、RPG、开放世界等玩法，也不代表已确认制作预算。

### 节日活动、结婚、昼夜循环等具体特性

有些要求不是固定的 Steam 分类，例如“能过节”“可以结婚”“有昼夜循环”。插件会根据 Steam 标签、类型和商店描述检查这些具体特性，但商店页没有写明的隐藏机制仍可能无法确认。

写成“必须”“一定要”“只接受”等明确强制措辞时，在检查正常完成的情况下，确认不符或资料不足的游戏会被移除；服务暂时异常时则会保留并标注未确认。普通核心要求会优先展示已确认满足的游戏，资料未知的游戏只可能在结果不足时补位；“最好有”等可选要求只会让已确认满足的游戏更靠前。

### 开发商和发行商

```text
/gamerec FromSoftware 开发的单机动作游戏
/gamerec Capcom 发行的合作游戏
/gamerec 最好是 Valve 的游戏
```

插件会依据 Steam 公布的开发商和发行商信息判断。即使写了“仅限”或“只要”，公司条件目前仍会影响排序而不是直接排除，资料未知或不匹配时会降低优先级。

### 语言、预算和地区

```text
/gamerec 必须支持简体中文的殖民模拟，预算 100 元
/gamerec -US online co-op under $30
```

语言、预算和公司条件会在玩法相近的游戏之间影响排序。使用“必须”“只接受”等措辞会带来更明显的影响，但资料未知或不满足的游戏仍可能保留，并在理由中说明。

未写币种时，预算按本次所选地区的常用币种理解。价格资料不足或价格插件不可用时，插件不会假装游戏符合预算，而会保留不确定提示并继续返回玩法合适的游戏。

### 未发售游戏

普通查询默认排除未发售游戏。只有明确写出“未发售”“即将发售”“待发售”“upcoming”等意图时，它们才会参与推荐。

未发售游戏已有真实评测时会使用真实评测；没有评测时只会采用中性的默认值参与排序，并明确提示资料不足，不会把默认排序值显示成玩家评价。

## 怎样理解结果

推荐可能以合并转发或普通文本显示，显示样式取决于当前聊天平台。总览会说明找到多少款游戏，每款游戏会单独展示推荐分、理由、可用价格和 Steam 链接。

示例：

```text
找到 3 款 Steam 游戏，按核心匹配和推荐分排列。

1. 《示例游戏》｜推荐分：86/100

推荐理由：命中合作与解谜核心特征，Steam 评测基础较稳。

不推荐理由：简体中文支持信息未知。

价格（US）：当前价 $19.99；历史最低 $9.99；最近促销 $11.99

购买链接：https://store.steampowered.com/app/123456/
```

- `推荐分` 只用于比较本次结果，不代表“有 86% 的概率会喜欢”。
- 核心玩法是否匹配最重要；玩法相近时，再参考口碑、关注度、语言、公司和预算等偏好。
- `推荐理由` 说明适合你的地方；存在重要不符或资料不确定时，才会显示 `不推荐理由`。
- 多条提示会逐行列出，方便区分名称歧义、游戏库过滤和 Steam 查询异常。
- 同一作品的多个标准版、重制版或特别版通常只保留一个，避免结果被不同版本占满；正常续作仍会分别推荐。

## 账号、游戏库与价格

### 绑定账号

```text
/accountbind 76561198000000000
/accountbind 123456789
/accountbind
```

前两种分别可填写 17 位 SteamID64 或纯数字好友码；不带参数时查看当前聊天用户的绑定。绑定本身不要求 Steam Web API Key。

普通推荐不需要 Steam Web API Key。以下功能需要 Key、已绑定账号以及公开可读的 Steam 游戏库：

- `排除已有`：排除已拥有游戏及其同作品版本。
- `仅查看已有`：只推荐游戏库中已拥有的游戏。
- `/randomrec`：从未玩且评价达到 Dashboard 门槛的库内游戏中随机选择一款。

`/randomrec` 返回游戏名和简短理由，不附带推荐分、价格或购买链接。

### 区域价格

安装“Steam 价格查询（小黑盒）”（`astrbot_plugin_steam_price_heybox`）后，可以显示本次地区的当前价、历史最低和最近促销。缺少价格或促销数据时会显示“暂无数据”，但不会阻止游戏推荐。

价格只反映查询时可获得的信息，不提供跨区购买建议，也不保证未来售价或促销安排。

## 错误、降级与空结果

- 如果看到“偏好解析模型暂时不可用”，插件会继续推荐，但复杂条件可能理解不完整。
- 如果看到具体玩法暂时无法确认，相关游戏会保留不确定提示，不会把服务异常当成游戏明确不满足要求。
- Steam 部分数据暂时不可用时，插件会尽量使用仍可获得的资料继续；无法完成查询时会提示稍后重试。
- “参考游戏未能可靠解析”表示名称没有可靠命中或存在歧义，可以改用完整中文名、英文名或 Steam 商店标题重试。
- 当 Steam 查询正常但没有合格候选时，会返回“暂时没有找到满足当前条件的游戏。”可以尝试放宽核心玩法、排除项、人数或具体特性。

无结果灵感推荐默认关闭。启用后，只会在 Steam 查询正常但没有合适结果时出现；Steam 查询失败、程序错误或已经找到正式结果时不会使用。

灵感结果会单独标明“需求匹配未验证”，表示游戏名称已经确认存在，但是否符合需求尚未验证。它不会附带购买链接、价格、评测或推荐分，也不能用于“换一批”；模型不可用或名称无法确认时，仍会保留原本的空结果说明。

## 数据边界

- 正式推荐只包含 Steam 游戏；主动开启的无结果灵感会与正式结果明确分开。
- DLC、试玩版、原声、工具和套餐不会进入正式结果，无法确认类型时也不会猜测。
- 语言、开发商、发行商、发售状态、评测和商店描述以 Steam 当前公开信息为准；资料缺失会显示为未知。
- Steam 社区标签可能缺失、变化或包含玩梗内容，因此公开资料无法保证始终准确。
- 具体玩法只依据 Steam 商店可见信息判断，不能替代攻略、实际游玩体验或开发者确认。
- “AAA/大作”只代表偏好知名作品，因为 Steam 没有可靠的制作预算信息。
- 推荐结果用于缩小选择范围，不代表购买保证；决定前仍建议查看 Steam 商店页、系统需求和近期玩家评价。

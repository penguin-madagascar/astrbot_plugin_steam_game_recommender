# astrbot_plugin_steam_game_recommender

## 项目定位

Steam 游戏推荐助手是一款面向 AstrBot 的自然语言推荐插件。它把玩家对玩法、人数、语言、预算和参考游戏的描述转换为一组按推荐分排序的 Steam 游戏，并给出简短理由、指定区域价格和 Steam 商店链接。

普通推荐无需 Steam Web API Key。绑定 Steam 账号后，插件还可以结合游戏库过滤结果，并从尚未游玩的游戏库中随机推荐下一款游戏。

## 核心能力

- 理解合作方式、玩法类型、难度、氛围、语言、预算、喜欢或不喜欢的参考游戏等自然语言偏好。
- 推荐候选的搜索条目必须是 `app`，详情类型必须是 `game`；DLC、Demo、原声、工具和套餐不会进入已验证推荐。
- 同一作品只保留一款：通常优先标准本体，没有标准本体时保留推荐分最高的完整版本。
- 按查询动态确定核心与辅助标签，结合参考游戏解析、游戏库画像、Steam 口碑和评测规模生成 0–100 的推荐分。
- 根据 Steam 商店语言信息核对语言要求，并分别识别简体中文与繁体中文。
- 按查询区域展示当前价格、历史最低和最近促销，购买入口只指向 Steam。
- 支持排除已拥有游戏、只推荐库内游戏，以及在近期推荐基础上补充条件或换一批。
- 可从已绑定游戏库中随机推荐一款尚未游玩且口碑达到要求的游戏。
- 推荐理由优先使用可验证的游戏信息；模型不可用或结果不可靠时自动使用规则化理由。空结果 LLM 兜底默认关闭，只有显式选择专用模型后才会启用。

## 快速开始

### 命令

| 命令 | 中文 alias | 用途 |
| --- | --- | --- |
| `/gamerec` | `/游戏推荐` | 按偏好推荐 Steam 游戏，可同时限定游戏库范围和价格区域。 |
| `/gamerec_retry [补充要求]` | `/重新推荐`、`/换一批` | 基于最近一次结果补充条件或更换候选。 |
| `/accountbind <SteamID64|好友码>` | `/账号绑定` | 绑定或查看当前聊天用户的 Steam 账号。 |
| `/randomrec` | `/随机推荐` | 从库内尚未游玩的游戏中随机推荐一款。 |

`/gamerec` 的完整格式为：

```text
/gamerec [排除已有|仅查看已有] [区域] <自然语言需求>
```

### `/gamerec` 组合用例

游戏库过滤参数必须位于需求开头，并且 `排除已有` 与 `仅查看已有` 不能同时使用。区域可以写成 `-CN`、`-US`、`-JP` 等两字母代码，也可以使用国区、美区、日区、港区、台区、韩区等常用名称。

```text
/gamerec 排除已有 -US 双人合作解谜，预算 30 美元
/游戏推荐 仅查看已有 日区 适合周末通关的剧情游戏，预算 3000 日元
/gamerec 排除已有 国区 支持简体中文的轻松经营游戏，预算 100 元
```

未指定区域时使用插件配置的默认区域；预算未写明币种时按所选区域的本币理解。游戏库过滤和 `/randomrec` 需要 Steam Web API Key、已绑定账号以及公开可读的 Steam 游戏库。

## 推荐结果

`/gamerec` 会先发送一句简短总览。有核心标签时，结果按核心匹配层级及层内推荐分排列；宽泛查询则直接按推荐分展示：

```text
找到 3 款 Steam 游戏，按核心匹配层级及层内推荐分排列。

1. 《示例游戏》｜推荐分：86/100
推荐理由：玩法契合合作解谜偏好。Steam 口碑稳定，但简体中文支持尚未确认。
价格（US）：当前价 $19.99；历史最低 $9.99；最近促销 $11.99（结束于 18 天前）
购买链接：https://store.steampowered.com/app/123456/
```

### 候选准入与版本去重

普通推荐、`/gamerec_retry` 和 `/randomrec` 都使用 Steam 结构化应用类型作为硬门槛。搜索结果只接受 `app`，详情只接受 `type == "game"`；详情失败、类型缺失或属于 DLC、Demo、原声、工具等其他[Steam 应用类型](https://partner.steamgames.com/doc/store/application?language=english)时会直接跳过。Steam 搜索中的 `sub` 和 `bundle` 套餐也会在搜索阶段排除。

候选排序后会按作品族去重。系统只剥离标题末尾明确的版本标记，例如 `Special Edition`、`Director's Cut`、`Remastered`、`VR`、特别版、终极版和重制版；续作编号和普通副标题不会被移除。参考游戏标题匹配会把同一编号的阿拉伯数字与罗马数字视为等价，同时拒绝把基础版或其他编号误解析为目标续作。普通推荐优先标准本体；没有标准本体时保留该作品族中推荐分最高的版本，并按所选版本自身的推荐分参与最终排序。

换一批和“排除已有”会排除整个作品族；可靠解析的参考游戏只排除精确种子 AppID，其续作、同系列和其他版本仍可参与推荐。“仅查看已有”仍按精确 AppID 匹配，并在同作候选中优先选择实际拥有的版本。`/randomrec` 会先压缩库内同作版本，再核对详情类型和评价门槛。

### 推荐分构成

`/gamerec` 使用固定的锚点分层策略，不提供可破坏层级约束的手工评分权重。用户明确要求的玩法和参考游戏中最具区分度的标签会成为核心锚点；用户游戏库画像只能补充辅助相似度，不能代替核心标签或硬条件。候选始终先按核心覆盖层级排序，再比较同层推荐分，因此仅匹配宽泛类型的高热度游戏不能越过更强相关的候选。

参考游戏会把中文名、英文名和本地化别名作为同一实体解析。系统先查询参考游戏前五个 Steam 有序标签的结果规模，再选择最多两个更具区分度的标签作为锚点；每个召回锚点保持 Steam 默认相关性排序并独立取候选，高知名度/大作倾向查询还会合入畅销榜。标签页与标签 ID 使用 24 小时新鲜缓存，网络异常时最多使用 7 天的原始快照；超过期限或接口契约失效后会改用标签文本搜索和本地索引补位，并在结果中提示召回能力下降。

对于“玩法 + AAA/大作”这类查询，AAA 只表示高知名度倾向，不会自动扩展为动作、RPG、开放世界等类型。偏好解析器如需把不同语言的玩法短语转换成 Steam 规范标签，必须同时提供可在用户原文中核验的最短连续片段；没有原文证据的类型扩展会被删除。

内部相关层级分为：A 层满足硬约束且核心覆盖率至少为 0.60；B 层满足硬约束、至少命中一个核心锚点但覆盖率低于 0.60；C 层满足硬约束但没有命中核心锚点，只在更高层候选不足时宽松补位。没有核心锚点的宽泛查询使用单一层级。

层内评分由以下固定公式计算：

```text
有核心锚点：语义分 = 70% × 核心覆盖率 + 30% × 辅助标签相似度
无核心锚点：语义分 = 辅助标签相似度
质量分 = 60% × Wilson 好评率下界（z=1.96）+ 40% × 评测量知名度
普通查询：层内分 = 70% × 语义分 + 30% × 质量分
高知名度/大作倾向：层内分 = 55% × 语义分 + 45% × 质量分
```

评测缺失或为零时质量分为 0，不再继承候选池平均口碑。负向参考相似度按 0.25 系数从语义分扣除；正向参考用于构建核心和辅助标签，不会再作为独立分量重复计分。推荐分最终四舍五入并限制在 0–100。

语言只在用户明确提及时生效，不参与核心标签覆盖，也不会跨越 A/B/C 相关层级。候选支持全部所需语言时软调整为 0；Steam 语言数据未知时为 -2；普通语言偏好不支持时为 -5；使用“必须”“一定要”“只接受”、`must` 或 `required` 等强制措辞而不支持时为 -10。一次提出多种语言时只采用其中最严重的一次调整，并在推荐证据中明确提示。

预算同样不会淘汰候选，只会在同一 A/B/C 层内重新排序，不会跨越相关层级。当前价不高于预算时为 +5；仅历史最低不高于预算时为 0；价格数据不足、币种不可比较或价格插件不可用时为 -2；当前价和历史最低均已知且都超出预算时，普通预算为 -5，明确使用强制措辞的预算为 -10。

价格始终对应本次查询区域；价格插件不可用或数据缺失时，推荐本身仍可正常返回。`/randomrec` 使用普通单条消息，只提供游戏名和简短推荐理由，不附带分数、价格或链接。

## 安装与配置

1. 通过 AstrBot 插件管理安装，或将项目目录以 `astrbot_plugin_steam_game_recommender` 为名放入 AstrBot 插件目录。
2. 安装项目依赖：

   ```bash
   pip install -r requirements.txt
   ```

3. 直接使用 `/gamerec`；普通推荐不要求 Steam Web API Key。

Dashboard 配置按用途整理为四个卡片：

| 配置卡片 | 内容 |
| --- | --- |
| 模型与鉴权 | 选择用于偏好解析和推荐理由的 LLM、为空结果兜底显式选择专用模型，以及填写 Steam Web API Key。可从 [Steam 官方申请页](https://steamcommunity.com/dev/apikey) 获取 Key。 |
| 价格与地区 | 查看 Steam 价格增强说明，并设置未指定区域时使用的默认地区。 |
| 推荐与评分 | 设置推荐数量、索引缓存时间，以及仅供 `/randomrec` 使用的最低评测数量和好评率门槛；`/gamerec` 使用固定的锚点分层与 Wilson 质量公式。 |
| 缓存与网络 | 设置 Steam 数据查询缓存和网络请求超时。 |

`llm_fallback_provider_id` 默认为空，即不启用空结果 LLM 兜底。显式选择模型后，只有 Steam 索引或过滤结果为空（包括明确的非 Steam 平台请求）时才会调用它；输出以“⚠️ LLM 兜底建议（未经过 Steam 索引验证）”标注。这类建议没有经过 Steam 应用类型、同作版本或套餐校验，插件只会要求模型尽量避开 DLC、原声、工具、套餐和不同版本，无法提供硬保证。

从旧版本升级时，旧版平铺配置会在首次加载时自动迁移到这四个卡片，已有的常规模型、Steam Key、地区和推荐参数会保留。旧的五项评分权重会被清除；旧的空结果兜底布尔字段无论开关状态都会删除，`llm_fallback_provider_id` 会重置为空，避免升级后意外启用未验证推荐。

如需区域价格与促销信息，请同时安装 `astrbot_plugin_steam_price_heybox`；该价格插件无需在本插件内重复配置。

## 使用边界

- 已验证结果只来自搜索类型为 `app`、详情类型为 `game` 的 Steam 商店应用；详情未知时不会猜测为游戏本体。
- 插件默认不为非 Steam 平台生成候选。只有显式配置 `llm_fallback_provider_id` 时，才可能返回醒目标注且未经类型、版本和套餐校验的 LLM 兜底建议。
- 游戏库相关能力依赖有效的 Steam Web API Key、账号绑定和公开可读的个人资料。
- `/randomrec` 将 Steam 记录中游玩时长为零的游戏视为尚未游玩；它会先压缩同作版本，再从详情确认为游戏本体且评价达到配置要求的候选中随机选择。
- 区域价格来自可用的 Steam 价格数据，不提供跨区域比价，也不保证未来促销价格。
- 语言、人数、合作方式、难度和氛围取决于 Steam 商店信息与公开标签；数据缺失时会保留不确定性，不会推断为支持。
